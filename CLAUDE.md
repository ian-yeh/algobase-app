# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Algobase** is a speedcubing training app that tracks solve times and provides performance analytics. It's a single-page React app backed by Convex for both data and authentication.

### Architecture

**Frontend** (`/web`): React 19 + TypeScript + Vite
- Convex React client for queries, mutations, and live updates
- Zustand (`persist` middleware) for client-side auth state — see `src/stores/authStore.ts`
- React Router for navigation
- Tailwind CSS for styling
- Chart.js / `react-chartjs-2` for performance visualizations

**Backend** (`/convex`): Convex functions in TypeScript
- `schema.ts` — `users`, `passwords`, `solves` tables with indexes
- `auth.ts` — email/password sign-up + sign-in mutations; PBKDF2 hashing; signed HMAC tokens (7-day expiry)
- `solve.ts` — `getSolves`, `createSolve`, `deleteSolve`, `getStats`
- `user.ts` — `getUser`, `updateUser`
- All non-auth functions take a `token` arg and call `verifyToken` from `auth.ts` to authorize the caller

Convex hosts the database and runs functions as serverless queries/mutations. There is no separate API server.

## Commands

### Run everything (recommended)
```bash
npm run dev          # concurrently: vite dev server + convex dev (function push + watch)
```

### Frontend only
```bash
npm run web:dev      # Vite dev server with HMR
npm run web:build    # tsc -b && vite build
npm run web:lint     # ESLint
```

### Convex only
```bash
npm run convex:dev     # push functions to the dev deployment and watch for changes
npm run convex:deploy  # deploy to production
```

### Switch / configure a Convex deployment
```bash
npx convex dev --configure   # interactive: pick existing or create new
```
This rewrites `.env.local` (root) with `CONVEX_DEPLOYMENT`. Then also update `web/.env`'s `VITE_CONVEX_URL` to match — these two must point at the same deployment or the browser WebSocket will fail to connect.

## Authentication Flow

- Sign-up / sign-in mutations live in `convex/auth.ts`. They return `{ token, user }`.
- `SignIn.tsx` calls the mutation directly with `useMutation`, then dispatches the result to the Zustand store via `setAuth({ user, token })`.
- The store persists `{ user, token }` to `localStorage` under key `auth`. Page refresh hydrates synchronously — no loading flicker.
- Every protected query/mutation takes the `token` as an argument and calls `verifyToken` inside the Convex function. Tokens are signed HMAC-SHA256 with a 7-day expiry.
- Sign-out: `useAuthStore.getState().clearAuth()` wipes the store + localStorage entry. The sidebar's `LogOut` button does this and navigates to `/signin`.

## Environment Variables

### Root `.env.local` (read by Convex CLI)
- `CONVEX_DEPLOYMENT` — e.g., `dev:energized-fly-476`
- `CONVEX_URL` / `CONVEX_SITE_URL` — informational

### `/web/.env` (read by Vite, baked into the bundle)
- `VITE_CONVEX_URL` — full cloud URL of the Convex deployment (must match `.env.local`)

### Convex dashboard (set per deployment, not in repo)
- `CONVEX_JWT_SECRET` — HMAC secret for auth tokens. If unset, falls back to a hardcoded dev string — **set this before going to prod**.

## Common Tasks

- **Add a new field to a table**: edit `convex/schema.ts` → save (convex dev hot-reloads + applies the schema change). No migration files; Convex enforces the new shape going forward.
- **Add a new Convex function**: create or extend a file in `convex/` and export `query` / `mutation` / `action`. `convex dev` regenerates `_generated/api` automatically, then the frontend can call it via `useQuery(api.module.fn, ...)` or `useMutation(api.module.fn)`.
- **Add a protected function**: take a `token: v.string()` arg, call `await verifyToken(args.token)` first, throw on null. Mirror the pattern in `convex/solve.ts`.
- **Update frontend styling**: Tailwind via `web/src/index.css`; classes inline on components.

## Development Notes

- **Auth state is global via Zustand**, not Context. Read from any component with `useAuthStore((s) => s.token)`. Don't reintroduce `AuthContext`.
- **Token expiry is not yet handled gracefully.** When a token passes the 7-day mark, Convex calls throw `"Invalid token"` and screens will show errors instead of redirecting. Add a global handler before serious use.
- **Per-user password salt is not yet implemented** — `convex/auth.ts` uses a static salt. Fine for dev, needs replacement before prod.
- **No production deployment configured yet.** `npm run convex:deploy` exists but Vercel/Netlify/etc. setup for the frontend is not wired in.
